# 웹 네비게이션 구조 분석 스킬

## 개요
Chrome 브라우저로 웹 애플리케이션에 접속하여 GNB(상단 네비게이션)와 LNB(좌측 사이드바) 구조를 자동 분석하고, 각 메뉴별 화면 정보를 포함한 구조화된 Markdown 파일을 생성합니다.
이 파일은 `web-testcase-creator` 스킬의 입력으로 사용됩니다.

## 인자
- `url` (필수): 분석 대상 웹 애플리케이션 URL
- `output_path` (선택, 기본값: 현재 디렉토리에 `nav-structure.md`): 결과 MD 파일 저장 경로

## 실행 절차

### 1단계: 브라우저 초기화 및 접속
```
1. tabs_context_mcp으로 탭 컨텍스트 확인 (createIfEmpty: true)
2. 새 탭 생성 (tabs_create_mcp)
3. url로 navigate
4. SSL 인증서 경고가 나타나면:
   - JavaScript로 #details-button 클릭
   - #proceed-link 클릭하여 진행
5. 페이지 로드 대기 (wait 3초 + screenshot)
```

### 2단계: 로그인 필요 여부 확인
```
1. screenshot으로 현재 화면 캡처
2. 로그인 페이지인지 판단:
   - find로 "로그인", "Login", "Sign in" 버튼/텍스트 탐색
   - 계정/비밀번호 입력 필드 존재 여부 확인
3. 로그인이 필요한 경우:
   - AskUserQuestion으로 사용자에게 로그인 정보 요청:
     "로그인이 필요합니다. 계정과 비밀번호를 알려주세요."
   - 사용자가 제공한 정보로 로그인 수행
   - 로그인 성공 확인 (대시보드/메인 페이지 로드)
4. 로그인 불필요 시 바로 3단계 진행
```

### 3단계: GNB (Global Navigation Bar) 분석
```
1. screenshot으로 전체 화면 캡처
2. read_page로 접근성 트리의 상단 영역 분석:
   - <header>, <nav>, role="navigation", role="banner" 영역 탐색
   - 상단에 위치한 메뉴 링크/버튼 목록 수집
3. GNB 요소 식별:
   - 로고/브랜드 영역
   - 1차 메뉴 항목 (텍스트 + 링크)
   - 드롭다운/메가메뉴 존재 여부
   - 사용자 프로필/알림 등 유틸리티 영역
4. 각 GNB 메뉴에 hover 또는 클릭하여 하위 메뉴 확인:
   - computer의 hover 액션으로 드롭다운 트리거
   - screenshot + read_page로 하위 항목 수집
   - 하위 메뉴의 텍스트, 링크, 깊이 기록
```

### 4단계: LNB (Left Navigation Bar) 분석
```
1. read_page로 좌측 영역 분석:
   - <aside>, role="complementary", role="navigation" 중 좌측에 위치한 영역
   - 사이드바 메뉴 트리 구조 탐색
2. LNB 요소 식별:
   - 1차 메뉴 항목 (아이콘 + 텍스트)
   - 접혀있는(collapsed) 하위 메뉴 존재 여부
   - 현재 활성화된 메뉴 표시
3. 각 LNB 1차 메뉴 클릭하여 하위 메뉴 펼치기:
   - find로 메뉴 항목 찾기 → 클릭
   - wait 1초 → screenshot + read_page
   - 펼쳐진 하위 메뉴 항목 수집
   - 2depth, 3depth까지 재귀적으로 탐색
4. LNB가 없는 경우:
   - 탭 기반 네비게이션 확인
   - 드로어(drawer) 메뉴 확인
   - 해당 구조를 LNB 대신 기록
```

### 5단계: 메뉴별 화면 분석
각 메뉴(GNB + LNB의 최하위 메뉴)에 대해:
```
1. 해당 메뉴 클릭으로 페이지 이동
2. wait 2초 → screenshot으로 화면 캡처
3. read_page (filter: "interactive")로 인터랙티브 요소 수집:
   - 버튼: 텍스트, 위치 (상단/하단/인라인)
   - 입력 필드: 타입 (text/select/checkbox/radio/toggle), 라벨
   - 테이블/리스트: 컬럼명, 행 구조
   - 탭/패널: 탭 이름 목록
   - 모달 트리거: 다이얼로그를 여는 버튼
   - 검색/필터 영역
   - 페이지네이션
4. URL 패턴 기록
5. 화면 유형 분류:
   - 목록(List): 테이블/그리드 + CRUD 버튼
   - 상세(Detail): 단일 항목 정보 표시
   - 설정(Settings): 토글/입력 + 저장 버튼
   - 대시보드(Dashboard): 차트/통계/위젯
   - 폼(Form): 입력 필드 + 제출 버튼
```

### 6단계: MD 파일 생성
수집된 정보를 아래 형식의 Markdown 파일로 작성합니다.

**출력 형식:**
```markdown
# [애플리케이션 이름] 네비게이션 구조

> 분석 일시: YYYY-MM-DD HH:MM
> 대상 URL: [url]
> 로그인 필요: 예/아니오

## GNB (상단 네비게이션)

| 메뉴 | 하위 메뉴 | URL 패턴 |
|------|-----------|----------|
| 대시보드 | - | /dashboard |
| 관리 | 사용자 관리, 권한 관리 | /admin/* |

## LNB (좌측 사이드바)

```
메뉴1
  ├── 하위메뉴1-1
  │   ├── 하위메뉴1-1-1
  │   └── 하위메뉴1-1-2
  ├── 하위메뉴1-2
  └── 하위메뉴1-3
메뉴2
  └── 하위메뉴2-1
```

## 화면별 상세 분석

### [메뉴 경로] (예: 설정 > 사용자 관리)
- **URL**: /settings/users
- **화면 유형**: 목록(List)
- **주요 기능**:
  - 사용자 목록 테이블 (컬럼: 이름, 이메일, 역할, 상태)
  - 검색 필터 (이름, 이메일)
  - [+] 추가 버튼 → 생성 폼/모달
  - 행 클릭 → 상세/편집
  - 체크박스 선택 → 삭제 버튼
- **UI 요소 힌트**:
  - 추가 버튼: "추가" 또는 [+] 아이콘, 우측 상단
  - 검색 입력: placeholder "검색..."
  - 테이블: role="table" 또는 <table> 태그
  - 삭제 버튼: "삭제" 텍스트, 선택 시 활성화

### [다음 메뉴 경로]
...
```

### 7단계: 결과 보고
```
1. Write 도구로 output_path에 MD 파일 저장
2. 사용자에게 결과 요약 보고:
   - 총 분석된 메뉴 수 (GNB: N개, LNB: N개)
   - 총 분석된 화면 수
   - 파일 저장 위치
   - "web-testcase-creator 스킬에서 이 파일을 사용하여 테스트 케이스를 생성할 수 있습니다."
```

## 에러 처리
- **접속 불가**: 사용자에게 URL 및 네트워크 확인 요청
- **SSL 인증서 경고**: JavaScript로 자동 우회
- **로그인 실패**: 사용자에게 재입력 요청 (최대 3회)
- **메뉴 클릭 후 로드 실패**: 3초 대기 후 재시도, 실패 시 해당 메뉴 건너뛰고 기록
- **세션 만료**: 로그인 페이지 감지 시 재로그인 후 계속
- **SPA 라우팅**: URL 변경 없이 컨텐츠만 변경되는 경우 screenshot으로 변경 감지

## 주의사항
- 화면 분석 시 데이터를 변경하는 작업(생성, 수정, 삭제 버튼 클릭)은 절대 수행하지 않음
- 드롭다운/모달은 열어서 내용 확인 후 닫기 (ESC 키)
- 비밀번호는 MD 파일에 기록하지 않음
- 분석 중 발견한 팝업/배너는 닫고 계속 진행
- GNB/LNB가 명확히 구분되지 않는 경우 (탭 네비게이션, 햄버거 메뉴 등) 실제 구조에 맞게 유연하게 기록
